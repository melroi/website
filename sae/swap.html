Ajout d’un espace swap sur le serveur Debian


Pourquoi avons-nous ajouté du swap ?

Le serveur Debian dispose de seulement 4 Go de RAM DDR3, ce qui peut s’avérer insuffisant lorsqu’il exécute plusieurs services simultanément, comme le serveur FTP pour les transferts de fichiers, Apache2 pour le site web ou encore le serveur DHCP.

Dans ce contexte, l’ajout d’un fichier swap est essentiel. Le swap, ou mémoire virtuelle, agit comme une extension de la RAM sur le disque dur. Il permet de déplacer temporairement certaines données peu utilisées de la RAM vers le disque, libérant ainsi de l’espace pour les processus actifs.

Cela réduit fortement les risques de saturation de la mémoire, donc les ralentissements ou plantages du système, tout en assurant une meilleure stabilité du serveur.


Commandes utilisées

Nous avons créé un fichier swap de 8 Go sur la partition sda3 avec les commandes suivantes :

sudo fallocate -l 8G /swapfile

sudo chmod 600 /swapfile

sudo mkswap /swapfile

sudo swapon /swapfile

echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab


Pour que le système utilise ce swap de manière plus proactive, nous avons défini la valeur de swappiness à 70 :

sudo sysctl vm.swappiness=70

echo 'vm.swappiness=70' | sudo tee -a /etc/sysctl.conf
